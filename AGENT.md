# AGENT.md

## 当前背景

当前已存在的核心模块划分如下：

- `lib/main.ts`：插件主线程，负责选区校验、消息分发、调用业务逻辑、把结果落到画布
- `lib/blend.ts`：混合算法与插值逻辑
- `messages/sender.ts`：主线程与 UI 的消息协议与收发封装
- `ui/App.vue`：插件 UI 入口，只负责页面装配
- `ui/composables/useBlendTool.ts`：UI 状态、消息收发与预览态切换
- `ui/composables/useToolTabs.ts`：tab 状态与页面映射
- `ui/components/*`：UI 的结构组件，分别负责标题、tab、输入、状态和按钮
- `ui/pages/*`：页面级组件，分别承载混合、循环、复制 & 旋转页面

## 工作规则

1. 每次启动任务前，先读 `PLAN.md`，确认当前背景、进度和下一步。
2. 每轮任务结束后，必须更新 `PLAN.md`，把本轮完成内容、剩余工作和风险写进去。
3. 新功能必须按职责拆分到多个文件里写，避免把所有代码堆进一个文件。
4. 主线程、混合算法、消息协议、UI 必须保持分层，避免相互耦合。
5. 如果任务目标或实现方向发生变化，先更新 `PLAN.md` 再继续实现。

## 约束

- 以 MasterGo 插件 API 和当前工程结构为准，不假设浏览器页面能力可以直接使用。
- 画布相关操作只在主线程执行。
- UI 只负责交互，不直接操作画布节点。
- 网页预览和插件宿主要分开处理，浏览器里只做预览态，不假设存在 MasterGo 的 `mg` 宿主。
- 外部记忆只记录任务背景、当前进度和恢复步骤，不承担业务代码存储。

## 恢复步骤

1. 先读 `PLAN.md`
2. 理解当前状态与未完成项
3. 继续按照模块分工推进
4. 任务结束时回写 `PLAN.md`
